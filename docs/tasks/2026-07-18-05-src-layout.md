# TASK: Housekeeping — Move Sources into `src/`, Tag v2.1.0

## Goal

Move all Swift sources and asset resources out of repo root into
`src/`, so root contains only docs/config: `README.md`, `LICENSE`,
`build.sh`, `AGENTS.md`, `CLAUDE.md`, `.gitignore`, plus the `docs/`
directory. Then tag the result as `v2.1.0` (Phase 1 UI refactor +
Phase 2 settings parity).

**No behavior changes.** This is a pure file-move + build-script update.

## Current root contents (files to relocate)

Swift files (14, root level):
`AppDelegate.swift`, `AppState.swift`, `AppState+Dependencies.swift`,
`AppState+Export.swift`, `AppState+Generation.swift`,
`AppState+Loading.swift`, `AppState+Persistence.swift`,
`AppState+Sizing.swift`, `ContactSheetRenderer.swift`,
`Extensions.swift`, `FontPanelBridge.swift`, `FrameSheetApp.swift`,
`Models.swift`, `Thumbnail.swift`

Directories:
- `Views/` (already contains subfolders from Phase 1 — `Views/Tabs/`,
  `Views/Components/`, etc. per the Phase 1 split)
- `assets/`

Stays at root: `README.md`, `LICENSE`, `build.sh`, `AGENTS.md`,
`CLAUDE.md`, `.gitignore`, `docs/`.

## Steps

### 1. Move sources

- `mkdir -p src`
- `git mv` each of the 14 root `.swift` files into `src/`.
- `git mv Views src/Views`.
- Decide `assets/` placement: if these are Swift-compiled resources
  (e.g. referenced via `Bundle.module` or copied by `build.sh`), move
  to `src/assets/`; if they're purely doc/marketing assets (README
  screenshots etc.), leave at root or move to `docs/assets/` instead —
  inspect usage before moving and note the decision.

### 2. Update `build.sh`

- Update all source paths/globs to point at `src/` (e.g.
  `src/*.swift src/Views/**/*.swift`, or whatever glob/file-list
  mechanism the script uses).
- Update any output-path or resource-copy references touched by the
  `assets/` move.
- No change to compiler flags, target, or produced binary name.

### 3. Update cross-references

- Grep root and `docs/` for paths to the moved files/dirs (README,
  AGENTS.md, CLAUDE.md, docs/ARCHITECTURE.md, docs/DECISIONS.md, any
  `.github/` workflow if present) and update them to the new `src/`
  paths.

### 4. Verification

- `./build.sh` succeeds; `build/FrameSheet.app` produced, no new
  warnings.
- Byte-diff harness (from Phase 1/2) confirms identical output at
  default settings — this move must not touch generation behavior.
- `git log --follow src/AppState.swift` (and one or two others) shows
  continuous history through the Phase 1 split and original commits.
- `ls` at repo root matches the target list above exactly (only docs/
  config + `src/`).

### 5. Wrap-up + release

- `docs/DEV_LOG.md` entry for the housekeeping move.
- Archive this task to `docs/tasks/`.
- After merge to `main`: tag `v2.1.0` on the merge commit, with a tag
  message summarizing Phase 1 (structural refactor + single-column
  sidebar) and Phase 2 (settings-panel parity) as the release content.
  Do not include this housekeeping move itself as release content in
  the tag message — it's structural only.

## Constraints

- No Swift logic changes; `git mv` only for relocations.
- Follow `AGENTS.md` / `CLAUDE.md`. Branch off current `main`.
